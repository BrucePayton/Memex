# 外部资源管理功能设计与实现

## 概述
外部资源管理功能允许系统集成和管理各类外部资源，包括外部Agent、MCP服务器、Skill、工作流等，同时支持对接外部平台如Dify、DSC、Skill Hub等，实现资源的自动发现和同步。

## 数据模型设计

### 1. 外部资源注册表 (`t_external_resource_registration`)
用于存储各类外部资源的注册信息：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键，自增 |
| kind | String(32) | 资源类型：external_agent / mcp_server / skill / workflow |
| name | String(255) | 资源展示名称 |
| description | Text | 资源描述 |
| config | JSONB | 协议相关配置 |
| credentials_ref | String(255) | 凭证引用（不存储明文） |
| is_active | Boolean | 是否启用，默认True |
| platform_registration_id | Integer | 关联的平台注册ID（手动注册时为NULL） |
| external_id | String(255) | 平台侧原生资源ID |
| gmt_created | TIMESTAMP | 创建时间 |
| gmt_modified | TIMESTAMP | 修改时间 |
| gmt_deleted | TIMESTAMP | 删除时间（软删除） |

### 2. 平台注册表 (`t_platform_registration`)
用于存储外部平台的注册信息：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键，自增 |
| platform_type | String(64) | 平台类型：dify / dsc / skill_hub 等 |
| name | String(255) | 平台展示名称 |
| base_url | String(512) | 平台API基地址 |
| api_key_ref | String(255) | API密钥引用（不存储明文） |
| config | JSONB | 平台级扩展配置 |
| is_active | Boolean | 是否启用，默认True |
| gmt_created | TIMESTAMP | 创建时间 |
| gmt_modified | TIMESTAMP | 修改时间 |
| gmt_deleted | TIMESTAMP | 删除时间（软删除） |

## API接口设计

### 外部资源管理接口

#### 1. 获取外部资源列表
```http
GET /wp/external_resources?kind={resource_kind}
```
**参数**：
- `kind`：可选，按资源类型过滤

**返回**：
```json
{
  "success": true,
  "data": {
    "items": [
      {
        "id": 1,
        "kind": "external_agent",
        "name": "Dify数据分析Agent",
        "description": "用于业务数据分析的外部Agent",
        "config": {"endpoint": "/api/v1/chat"},
        "credentials_ref": "vault:dify_api_key",
        "is_active": true,
        "platform_registration_id": 1,
        "external_id": "app-123456",
        "gmt_created": "2024-01-01 12:00:00",
        "gmt_modified": "2024-01-01 12:00:00"
      }
    ]
  }
}
```

#### 2. 创建外部资源
```http
POST /wp/external_resources
```
**请求体**：
```json
{
  "kind": "external_agent",
  "name": "新的外部Agent",
  "description": "Agent描述",
  "config": {},
  "credentials_ref": "vault:api_key",
  "platform_registration_id": 1,
  "external_id": "app-7890"
}
```

#### 3. 更新外部资源
```http
PATCH /wp/external_resources/{resource_id}
```
**请求体**：支持部分字段更新
```json
{
  "name": "更新后的名称",
  "is_active": false
}
```

### 平台管理接口

#### 1. 获取平台列表
```http
GET /wp/platforms?platform_type={type}
```
**参数**：
- `platform_type`：可选，按平台类型过滤

#### 2. 创建平台
```http
POST /wp/platforms
```
**请求体**：
```json
{
  "platform_type": "dify",
  "name": "公司Dify平台",
  "base_url": "https://dify.example.com/v1",
  "api_key_ref": "vault:dify_master_key",
  "config": {}
}
```

#### 3. 更新平台
```http
PATCH /wp/platforms/{platform_id}
```

#### 4. 删除平台（软删除）
```http
DELETE /wp/platforms/{platform_id}
```

#### 5. 触发平台资源发现
```http
POST /wp/platforms/{platform_id}/discover
```
**请求体**：
```json
{
  "kind": "external_agent" // 可选，指定发现的资源类型
}
```
**功能**：
- 调用平台API获取可用资源列表
- 自动创建或更新对应的ExternalResourceRegistration记录
- 返回发现结果：新增、更新、跳过的资源数量

## 功能流程

### 1. 手动注册外部资源
1. 用户在前端填写资源信息
2. 调用创建外部资源API
3. 后端存储到ExternalResourceRegistration表
4. 资源可在系统中被引用和使用

### 2. 平台对接与自动发现
1. 管理员注册外部平台，填写连接信息
2. 触发资源发现功能
3. 后端调用平台API获取可用资源列表
4. 对比已有资源，自动创建新资源或更新已有资源
5. 返回发现结果给用户确认

### 3. 资源使用
- 外部Agent可在分析流程中作为节点被调用
- MCP服务器提供的工具可在Agent执行过程中被使用
- Skill可作为独立功能模块被集成到工作流中
- 外部工作流可被系统调用执行特定任务

## 安全设计
- 所有凭证不存储明文，仅存储密钥管理系统的引用
- API调用使用HTTPS加密传输
- 平台访问权限严格控制，仅授权用户可管理平台配置
- 所有资源访问操作记录审计日志

## 扩展能力
- 支持新增平台类型，只需实现对应平台的发现适配器
- 支持新增资源类型，扩展kind字段即可
- 配置字段使用JSONB类型，支持灵活的扩展配置
