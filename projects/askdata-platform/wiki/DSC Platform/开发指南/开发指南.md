# 开发指南

本文档面向需要在DSC Platform基础上进行二次开发、功能扩展和定制化开发的开发者，介绍项目的代码结构、开发规范、扩展方式等内容。

## 开发环境准备

### 必备工具
- Python 3.11+
- Git
- VS Code / PyCharm（推荐）
- Docker（用于本地启动依赖服务）
- Node.js 18+（前端开发需要）

### 代码风格工具
- `black`：Python代码格式化
- `flake8`：Python代码检查
- `mypy`：Python类型检查
- `pre-commit`：Git提交前钩子

### 安装开发依赖
```bash
# 安装基础依赖
pip install -r requirements.txt

# 安装开发依赖
pip install -r requirements-dev.txt

# 安装pre-commit钩子
pre-commit install
```

## 核心设计理念

### 1. 插件化设计
所有功能模块都采用插件化设计，支持动态扩展和热插拔：
- 节点组件可以通过注册机制添加
- 嵌入模型可以通过实现统一接口扩展
- 文件解析器可以按需添加
- 检索后处理钩子可以自定义

### 2. 接口抽象
核心功能都定义了抽象基类，依赖抽象而不是具体实现：
- `BaseNode`：所有流程节点的基类
- `BaseEmbedding`：所有嵌入模型的基类
- `BaseChunker`：所有分块策略的基类
- `BaseStorage`：所有文件存储的基类

### 3. 配置驱动
所有配置都通过环境变量管理，不需要修改代码即可调整功能：
- 不同环境使用不同的配置文件
- 配置项有合理的默认值
- 敏感配置支持从密钥管理系统获取

### 4. 异步优先
IO密集型操作都采用异步实现，提高并发性能：
- 所有API接口都是异步的
- 数据库操作使用异步驱动
- 文件处理和网络请求都是异步的

## 开发规范

### 代码风格
- 遵循PEP 8规范
- 使用Black进行代码格式化，行宽120字符
- 使用Flake8进行代码检查
- 使用Mypy进行静态类型检查
- 函数和类需要有清晰的类型注解

### 命名规范
- 文件名：小写+下划线，如 `agent_flow_node.py`
- 类名：大驼峰，如 `KnowledgeEmbedding`
- 函数名：小写+下划线，如 `chunk_document`
- 变量名：小写+下划线，如 `chunk_size`
- 常量名：大写+下划线，如 `MAX_FILE_SIZE`

### 注释规范
- 公共API需要有详细的文档字符串，说明功能、参数、返回值、异常
- 复杂的业务逻辑需要有注释说明
- 避免无用的注释，代码本身应该尽量自解释
- TODO注释需要包含作者和预计完成时间

### 提交规范
Git提交信息遵循Conventional Commits规范：
```
<类型>(<范围>): <主题>

<正文>

<页脚>
```

类型说明：
- `feat`：新功能
- `fix`：修复bug
- `docs`：文档更新
- `style`：代码格式调整，不影响逻辑
- `refactor`：重构，不新增功能也不修复bug
- `perf`：性能优化
- `test`：测试相关
- `chore`：构建/工具/依赖等调整

示例：
```
feat(agent_flow): 新增自定义节点注册功能

- 添加节点自动发现机制
- 支持节点元数据配置
- 完善节点权限控制

Closes #123
```

## 常见开发场景

### 场景1：自定义流程节点开发
流程节点是扩展平台功能最常见的方式，以下是开发自定义节点的步骤。

#### 步骤1：创建节点类
```python
# 在agent_flow/agent_flow_nodes/目录下创建custom_node.py
from typing import Dict, Any
from pydantic import Field
from agent_flow.agent_flow_node import BaseNode
from agent_flow.agent_flow_base.agent_flow_base_task import TaskContext
from agent_flow.agent_flow_config.agent_flow_category_config import NodeCategory, NodeType

class CustomDataAnalysisNode(BaseNode):
    """自定义数据分析节点"""
    
    # 节点元数据
    node_type: NodeType = "custom_data_analysis"
    node_name: str = "数据分析"
    category: NodeCategory = "data_processing"
    description: str = "对输入数据进行自定义分析处理"
    version: str = "1.0.0"
    
    # 节点参数定义
    class Parameters(BaseNode.Parameters):
        analysis_type: str = Field(
            ..., 
            description="分析类型",
            options=["统计分析", "趋势预测", "异常检测"]
        )
        output_format: str = Field(
            default="json", 
            description="输出格式",
            options=["json", "csv", "markdown"]
        )
        custom_config: Dict[str, Any] = Field(
            default_factory=dict,
            description="自定义配置"
        )
    
    # 节点输入定义
    class Inputs(BaseNode.Inputs):
        input_data: Any = Field(..., description="输入数据")
    
    # 节点输出定义
    class Outputs(BaseNode.Outputs):
        analysis_result: Any = Field(..., description="分析结果")
        statistics: Dict[str, Any] = Field(..., description="统计信息")
    
    async def execute(self, task_context: TaskContext) -> Outputs:
        """节点执行逻辑"""
        # 获取参数
        params = task_context.parameters
        input_data = task_context.inputs["input_data"]
        
        # 业务逻辑实现
        analysis_result = await self._perform_analysis(input_data, params)
        statistics = self._calculate_statistics(analysis_result)
        
        # 返回结果
        return self.Outputs(
            analysis_result=analysis_result,
            statistics=statistics
        )
    
    async def _perform_analysis(self, data: Any, params: Dict) -> Any:
        """执行实际的分析逻辑"""
        # 实现自定义分析逻辑
        analysis_type = params["analysis_type"]
        
        if analysis_type == "统计分析":
            return self._statistical_analysis(data)
        elif analysis_type == "趋势预测":
            return self._trend_prediction(data)
        elif analysis_type == "异常检测":
            return self._anomaly_detection(data)
        else:
            raise ValueError(f"不支持的分析类型: {analysis_type}")
    
    def _calculate_statistics(self, result: Any) -> Dict:
        """计算统计信息"""
        return {
            "process_time": "2024-01-01T12:00:00Z",
            "record_count": len(result) if isinstance(result, list) else 1,
            "success": True
        }
```

### 场景2：自定义嵌入模型扩展
如果需要支持新的嵌入模型，可以按照以下步骤扩展。

#### 步骤1：实现嵌入模型类
```python
# 在vectors_mgt/embedding_providers/目录下创建custom_provider.py
from typing import List, Dict
from vectors_mgt.knowledge_vectors_basic import BaseEmbedding
from vectors_mgt.vector_config import VectorConfig

class CustomEmbeddingProvider(BaseEmbedding):
    """自定义嵌入模型提供商"""
    
    def __init__(self, config: VectorConfig):
        super().__init__(config)
        self.api_key = config.embedding_api_key
        self.api_endpoint = config.embedding_api_endpoint or "https://api.custom.com/embeddings"
        self.model_name = config.embedding_model or "custom-embedding-v1"
        self.dimension = config.embedding_dimension or 1024
    
    async def embed_texts(self, texts: List[str]) -> List[List[float]]:
        """批量生成文本向量"""
        # 实现调用自定义嵌入服务的逻辑
        import aiohttp
        
        headers = {
            "Authorization": f"Bearer {self.api_key}",
            "Content-Type": "application/json"
        }
        
        payload = {
            "model": self.model_name,
            "input": texts,
            "encoding_format": "float"
        }
        
        async with aiohttp.ClientSession() as session:
            async with session.post(
                self.api_endpoint,
                headers=headers,
                json=payload,
                timeout=30
            ) as response:
                response.raise_for_status()
                result = await response.json()
                
                # 提取向量结果
                embeddings = [item["embedding"] for item in result["data"]]
                return embeddings
    
    async def embed_query(self, query: str) -> List[float]:
        """生成查询向量（有些模型对查询有特殊处理）"""
        return (await self.embed_texts([query]))[0]
    
    def get_dimension(self) -> int:
        """返回向量维度"""
        return self.dimension
```

#### 步骤1：定义Pydantic模型
```python
# 在对应模块的api_models.py中定义请求响应模型
from pydantic import BaseModel, Field
from typing import List, Dict, Any, Optional

class CustomAnalysisRequest(BaseModel):
    """自定义分析请求"""
    dataset_id: str = Field(..., description="数据集ID")
    query: str = Field(..., description="查询内容")
    analysis_params: Dict[str, Any] = Field(default_factory=dict, description="分析参数")

class CustomAnalysisResponse(BaseModel):
    """自定义分析响应"""
    result: Any = Field(..., description="分析结果")
    related_documents: List[Dict] = Field(default_factory=list, description="相关文档")
    process_time: float = Field(..., description="处理时间，单位秒")
```

#### 步骤2：实现API逻辑
在对应模块的api_helper.py中实现业务逻辑：
```python
from typing import Dict, Any
from utils.logger_utils import logger
from knowledge_mgt.knowledge_api.api_helper import KnowledgeProcessHelper

async def perform_custom_analysis(
    dataset_id: str,
    query: str,
    analysis_params: Dict[str, Any]
) -> Dict[str, Any]:
    """执行自定义分析"""
    import time
    start_time = time.time()
    
    # 1. 检索相关文档
    related_docs = await KnowledgeProcessHelper.search_documents(
        dataset_id=dataset_id,
        query=query,
        limit=analysis_params.get("limit", 5)
    )
    
    # 2. 执行自定义分析逻辑
    result = await _run_custom_analysis(query, related_docs, analysis_params)
    
    process_time = time.time() - start_time
    
    return {
        "result": result,
        "related_documents": related_docs,
        "process_time": process_time
    }

async def _run_custom_analysis(query: str, docs: List, params: Dict) -> Any:
    """执行实际的分析逻辑"""
    # 实现自定义分析逻辑
    return {
        "query": query,
        "summary": "分析结果摘要",
        "details": [{"doc_id": doc["doc_id"], "score": doc["score"]} for doc in docs]
    }
```

#### 步骤3：添加API路由

```python
from fastapi import APIRouter, Depends, HTTPException
from typing import Dict, Any
from common_mgt.api_response import success, error
from .api_models import CustomAnalysisRequest, CustomAnalysisResponse
from .api_helper import perform_custom_analysis

router = APIRouter(prefix="/custom", tags=["custom"])

@router.post("/analysis", response_model=CustomAnalysisResponse)
async def custom_analysis(request: CustomAnalysisRequest):
    """自定义分析接口"""
    try:
        result = await perform_custom_analysis(
            dataset_id=request.dataset_id,
            query=request.query,
            analysis_params=request.analysis_params

        return success(result)
    except Exception as e:
        logger.error(f"自定义分析失败: {str(e)}", exc_info=True)
        return error(message=f"分析失败: {str(e)}", code=40000)
```

## 性能优化建议

### 1. 异步IO
- IO密集型操作一定要使用异步实现
- 避免在异步函数中调用同步IO操作，如果必须调用，使用`asyncio.to_thread`
- 数据库操作使用异步驱动，如asyncpg、aiomysql

### 2. 缓存策略
- 对高频访问的数据进行缓存，如配置信息、元数据
- 对重复计算的结果进行缓存，如向量嵌入、解析结果
- 使用多级缓存：内存缓存+Redis缓存

### 3. 批量处理
- 尽量使用批量操作，减少IO次数
- 数据库批量插入、批量更新
- 向量批量嵌入、批量入库

### 4. 资源隔离
- CPU密集型任务使用独立的进程池处理
- 不同优先级的任务使用不同的队列
- 关键业务和非关键业务资源隔离

## 常见问题排查

### 问题1：异步函数运行异常
- 检查是否在异步函数中调用了阻塞的同步函数
- 使用`asyncio.debug()`启用异步调试模式，查看是否有协程阻塞
- 检查是否正确使用了await关键字

### 问题2：数据库查询慢
- 查看慢查询日志，定位慢SQL
- 检查是否缺少必要的索引
- 对于复杂查询，考虑优化查询逻辑或添加缓存
- 大数据量查询考虑分页或异步处理

### 问题3：内存占用过高
- 检查是否有内存泄漏，尤其是全局变量和缓存
- 大文件处理时使用流式处理，避免一次性加载到内存
- 定期清理无用的缓存和临时文件
- 考虑使用对象池复用大对象

### 问题4：并发性能低
- 检查是否有串行处理的逻辑，可以优化为并行
- 数据库连接池大小是否足够
- Redis连接池配置是否合理
- 考虑水平扩展，增加服务节点
