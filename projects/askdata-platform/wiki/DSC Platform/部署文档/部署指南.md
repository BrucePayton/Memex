# 部署指南

本文档详细介绍DSC Platform的各种部署方式，包括本地开发部署、Docker单节点部署、生产环境集群部署等。

## 环境要求

### 最低配置
- CPU：2核
- 内存：4GB
- 磁盘：20GB可用空间
- 操作系统：Linux/Unix/macOS/Windows（推荐使用Linux）

### 推荐配置（生产环境）
- CPU：8核+
- 内存：16GB+
- 磁盘：SSD 100GB+
- 操作系统：Ubuntu 22.04 LTS / CentOS 8+

### 软件依赖
- Python 3.11
- Node.js 18+（前端部分）
- Redis 6.0+（缓存和消息队列）
- PostgreSQL 13+ / MySQL 8.0+（关系型数据库）
- Weaviate / Milvus（向量数据库）
- MinIO / 其他S3兼容存储（可选，文件存储）

## 本地开发部署

### 步骤1：获取代码
```bash
git clone https://github.com/your-org/dsc-platform.git
cd dsc-platform
```

### 步骤2：创建Python虚拟环境
```bash
# 使用conda
conda create -n dsc-platform python=3.11
conda activate dsc-platform

# 或者使用venv
python3.11 -m venv venv
source venv/bin/activate  # Linux/macOS
# venv\Scripts\activate  # Windows
```

### 步骤3：安装依赖
```bash
# 安装Python依赖
pip install -r requirements.txt

# 安装PaddleOCR（可选，如果需要OCR功能）
pip install paddlepaddle paddleocr
```

### 步骤4：配置环境变量
```bash
# 复制环境变量模板
cp .env.local.template .env

# 编辑.env文件，根据实际情况配置
vim .env
```

关键配置项说明：
```env
# 服务配置
API_HOST=0.0.0.0
API_PORT=5630
DEBUG=True

# 数据库配置
DATABASE_URL=postgresql://user:password@localhost:5432/dsc_platform
# 或者MySQL
# DATABASE_URL=mysql+pymysql://user:password@localhost:3306/dsc_platform

# Redis配置
REDIS_URL=redis://localhost:6379/0

# 向量数据库配置
VECTOR_DB_TYPE=weaviate
WEAVIATE_URL=http://localhost:8080
# 如果使用Milvus
# VECTOR_DB_TYPE=milvus
# MILVUS_URL=localhost:19530

# 嵌入模型配置
EMBEDDING_PROVIDER=openai
OPENAI_API_KEY=your-openai-api-key
OPENAI_BASE_URL=https://api.openai.com/v1
# 或者使用本地模型
# EMBEDDING_PROVIDER=local
# EMBEDDING_MODEL_PATH=./models/bge-large-zh-v1.5

# OCR配置
OCR_ENABLED=True
OCR_ENGINE=paddleocr

# 文件存储配置
STORAGE_TYPE=local
STORAGE_LOCAL_PATH=./upload_files
# 如果使用S3
# STORAGE_TYPE=s3
# S3_ENDPOINT=http://localhost:9000
# S3_ACCESS_KEY=minioadmin
# S3_SECRET_KEY=minioadmin
# S3_BUCKET=dsc-platform
```

### 步骤5：启动依赖服务
使用Docker快速启动依赖服务：
```bash
# 启动PostgreSQL
docker run -d \
  --name dsc-postgres \
  -e POSTGRES_USER=dsc \
  -e POSTGRES_PASSWORD=dsc123 \
  -e POSTGRES_DB=dsc_platform \
  -p 5432:5432 \
  postgres:15

# 启动Redis
docker run -d \
  --name dsc-redis \
  -p 6379:6379 \
  redis:7

# 启动Weaviate
docker run -d \
  --name dsc-weaviate \
  -e AUTHENTICATION_ANONYMOUS_ACCESS_ENABLED=true \
  -e PERSISTENCE_DATA_PATH=/var/lib/weaviate \
  -e DEFAULT_VECTORIZER_MODULE=none \
  -e ENABLE_MODULES='' \
  -p 8080:8080 \
  semitechnologies/weaviate:1.23.9
```

### 步骤6：初始化数据库
```bash
# 执行数据库迁移
alembic upgrade head

# 初始化基础数据
python scripts/init_data.py
```

### 步骤7：启动服务
```bash
# 方式1：直接启动
python app_run.py

# 方式2：使用uvicorn启动
uvicorn api_main:app --host 0.0.0.0 --port 5630 --reload

# 方式3：使用FastAPI CLI
fastapi run api_main.py --port 5630
```

### 步骤8：验证部署
访问以下地址验证服务是否正常：
- API文档：http://localhost:5630/docs
- 健康检查：http://localhost:5630/health

如果看到Swagger API文档页面，说明部署成功。

## Docker单节点部署

### 方式1：使用官方镜像
```bash
# 拉取镜像
docker pull dsc-platform/dsc-platform:latest

# 运行容器
docker run -d \
  --name dsc-platform \
  -p 5630:5630 \
  -e DATABASE_URL=postgresql://user:password@db-host:5432/dsc_platform \
  -e REDIS_URL=redis://redis-host:6379/0 \
  -e WEAVIATE_URL=http://weaviate-host:8080 \
  -e OPENAI_API_KEY=your-openai-api-key \
  -v ./upload_files:/app/upload_files \
  --restart always \
  dsc-platform/dsc-platform:latest
```

### 方式2：自行构建镜像
```bash
# 构建镜像
docker build -t dsc-platform:latest .

# 运行容器
docker run -d \
  --name dsc-platform \
  -p 5630:5630 \
  --env-file .env \
  -v ./upload_files:/app/upload_files \
  --restart always \
  dsc-platform:latest
```

### 使用Docker Compose部署全套环境
项目根目录提供了docker-compose.yml文件，可以一键启动所有依赖和服务：

```bash
# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

Docker Compose会自动启动以下服务：
- PostgreSQL 15
- Redis 7
- Weaviate 1.23.9
- MinIO（对象存储）
- DSC Platform主服务

## 生产环境集群部署

### 架构设计
生产环境推荐采用分布式部署架构：

```
┌─────────────────────────────────────────────────────┐
│                     负载均衡层                         │
│                     Nginx / ALB                       │
├─────────────────────────────────────────────────────┤
│                     应用服务层                         │
│              DSC Platform服务集群 × N                 │
├─────────────────────────────────────────────────────┤
│                     中间件层                          │
│        Redis集群 / Kafka集群 / Elasticsearch          │
├─────────────────────────────────────────────────────┤
│                     存储层                            │
│   PostgreSQL集群 / 向量数据库集群 / 对象存储集群       │
└─────────────────────────────────────────────────────┘
```

### 组件部署建议

#### 1. 负载均衡
- 推荐使用Nginx、云厂商ALB、F5等
- 配置HTTPS证书，启用HTTP/2
- 配置健康检查和自动故障转移
- 配置限流和WAF防护

#### 2. 应用服务
- 无状态设计，可以水平扩展
- 推荐Kubernetes部署，使用Deployment管理
- 配置资源限制：CPU 1-2核，内存 2-4GB
- 副本数根据业务量配置，至少2副本保证高可用
- 配置健康检查：`livenessProbe`和`readinessProbe`

#### 3. 数据库
- PostgreSQL/MySQL使用主从复制架构
- 配置定期备份和异地容灾
- 推荐使用云厂商托管的数据库服务
- 配置慢查询日志和性能监控

#### 4. 向量数据库
- Weaviate推荐至少3节点集群部署
- Milvus推荐分布式集群部署，根据数据量配置节点
- 配置数据持久化和定期备份
- 根据访问量配置资源，SSD存储

#### 5. 缓存和消息队列
- Redis使用集群模式或哨兵模式，保证高可用
- 数据持久化配置RDB+AOF
- 消息队列根据需要选择Kafka或RabbitMQ
- 配置消息持久化和死信队列

### Kubernetes部署示例

#### Deployment配置
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: dsc-platform
  labels:
    app: dsc-platform
spec:
  replicas: 3
  selector:
    matchLabels:
      app: dsc-platform
  template:
    metadata:
      labels:
        app: dsc-platform
    spec:
      containers:
      - name: dsc-platform
        image: dsc-platform/dsc-platform:latest
        ports:
        - containerPort: 5630
        env:
        - name: DATABASE_URL
          valueFrom:
            secretKeyRef:
              name: dsc-secrets
              key: database-url
        - name: REDIS_URL
          valueFrom:
            secretKeyRef:
              name: dsc-secrets
              key: redis-url
        - name: WEAVIATE_URL
          value: "http://weaviate:8080"
        - name: OPENAI_API_KEY
          valueFrom:
            secretKeyRef:
              name: dsc-secrets
              key: openai-api-key
        resources:
          requests:
            cpu: "1"
            memory: "2Gi"
          limits:
            cpu: "2"
            memory: "4Gi"
        livenessProbe:
          httpGet:
            path: /health
            port: 5630
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /health
            port: 5630
          initialDelaySeconds: 5
          periodSeconds: 5
        volumeMounts:
        - name: data
          mountPath: /app/upload_files
      volumes:
      - name: data
        persistentVolumeClaim:
          claimName: dsc-platform-pvc
```

#### Service配置
```yaml
apiVersion: v1
kind: Service
metadata:
  name: dsc-platform-service
spec:
  type: ClusterIP
  ports:
  - port: 80
    targetPort: 5630
  selector:
    app: dsc-platform
```

#### Ingress配置
```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: dsc-platform-ingress
  annotations:
    nginx.ingress.kubernetes.io/ssl-redirect: "true"
    nginx.ingress.kubernetes.io/proxy-body-size: "100m"
spec:
  tls:
  - hosts:
    - api.dsc-platform.com
    secretName: dsc-platform-tls
  rules:
  - host: api.dsc-platform.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: dsc-platform-service
            port:
              number: 80
```

## 配置详解

### 核心配置项

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `API_HOST` | 服务监听地址 | `0.0.0.0` |
| `API_PORT` | 服务监听端口 | `5630` |
| `DEBUG` | 是否开启调试模式 | `False` |
| `SECRET_KEY` | 加密密钥，生产环境必须修改 | 随机生成 |
| `DATABASE_URL` | 关系型数据库连接URL | - |
| `REDIS_URL` | Redis连接URL | `redis://localhost:6379/0` |
| `VECTOR_DB_TYPE` | 向量数据库类型，支持`weaviate`/`milvus`/`chroma` | `weaviate` |
| `WEAVIATE_URL` | Weaviate服务地址 | `http://localhost:8080` |
| `EMBEDDING_PROVIDER` | 嵌入模型提供商，支持`openai`/`qwen`/`zhipu`/`local` | `openai` |
| `OCR_ENABLED` | 是否启用OCR功能 | `True` |
| `STORAGE_TYPE` | 文件存储类型，支持`local`/`s3`/`minio` | `local` |
| `MAX_FILE_SIZE` | 最大上传文件大小，单位字节 | `104857600`（100MB） |
| `ALLOWED_FILE_TYPES` | 允许的文件类型 | 常见文档和图片格式 |

### 高级配置

#### 异步任务配置
```env
# Celery配置

```

## 监控和日志

### 日志配置
日志默认输出到控制台，也可以配置输出到文件：
```env
LOG_LEVEL=INFO
LOG_FILE=./logs/dsc-platform.log
LOG_MAX_SIZE=100MB
LOG_BACKUP_COUNT=10
```

### 监控指标
服务提供Prometheus监控指标：
```
http://your-server:5630/metrics
```

关键监控指标：
- API请求QPS和延迟
- 错误率统计
- 向量检索延迟和准确率
- 任务队列长度
- 资源使用率（CPU、内存、磁盘）

### 告警配置
推荐配置以下告警规则：
- 服务不可用告警
- 错误率>5%告警
- API延迟>3s告警
- 磁盘使用率>80%告警
- 内存使用率>85%告警
- 任务队列堆积告警

## 常见问题

### Q: 启动时提示缺少依赖？
A: 确保使用Python 3.11版本，并且所有依赖都安装成功。如果有依赖安装失败，可以尝试单独安装失败的包。

### Q: 数据库连接失败？
A: 检查数据库服务是否正常运行，配置的地址、端口、用户名、密码是否正确，网络是否连通。

### Q: 向量数据库连接失败？
A: 确认向量数据库服务已启动，配置的URL正确，并且没有防火墙或网络策略限制访问。

### Q: 文件上传失败？
A: 检查文件存储路径是否有写入权限，磁盘空间是否充足，文件大小是否超过限制。

### Q: OCR功能不生效？
A: 确认已安装PaddlePaddle和PaddleOCR，并且`OCR_ENABLED`配置为`True`。首次使用会自动下载模型，需要保证网络连通。

### Q: 嵌入模型调用失败？
A: 检查API Key是否正确，网络是否能访问对应的API服务，如果是本地模型，检查模型文件路径是否正确。

### Q: 如何升级版本？
A: 1. 备份数据库和文件
   2. 拉取新版本代码或镜像
   3. 执行数据库迁移：`alembic upgrade head`
   4. 重启服务
   5. 验证功能是否正常

### Q: 如何配置HTTPS？
A: 在负载均衡层配置HTTPS证书，或者在服务前使用Nginx反向代理并配置SSL证书。服务本身也支持直接配置SSL证书，通过环境变量`SSL_CERT_PATH`和`SSL_KEY_PATH`指定证书路径。
