# 外部资源管理模块架构

## 概述

外部资源管理模块是 AskData 平台集成外部服务（Dify / DSC / Skill Hub 等）的核心组件，分为**平台注册管理**、**资源注册管理**、**资源挂载管理**三个层级。[^src-2026-05-08-外部资源管理功能补全开发记录]

## 三层架构

```
┌─────────────────────────────────────────────────────┐
│               平台注册层 (Platform)                   │
│  t_platform_registration                            │
│  Dify / DSC / Skill Hub 等平台注册信息               │
│  ↓ 触发"资源发现"自动导入                             │
├─────────────────────────────────────────────────────┤
│               资源注册层 (External Resource)          │
│  t_external_resource_registration                   │
│  external_agent / mcp_server / skill / workflow     │
│  通过 platform_registration_id 关联到平台            │
│  ↓ 通过挂载引用                                       │
├─────────────────────────────────────────────────────┤
│               资源挂载层 (Workspace Mount)            │
│  t_workspace_resource_mount                         │
│  将资源挂载到具体工作空间                             │
│  支持关联关系（知识库 ↔ 数据库）                      │
└─────────────────────────────────────────────────────┘
```

## 数据模型

### PlatformRegistration (`t_platform_registration`)

外部平台注册表，存储 Dify/DSC/Skill Hub 等平台的连接信息。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| platform_type | String(64) | 平台类型：dify / dsc / skill_hub |
| name | String(255) | 展示名称 |
| base_url | String(512) | 平台 API 基地址 |
| api_key_ref | String(255) | 凭证引用（不存明文） |
| config | JSONB | 平台级扩展配置 |
| is_active | Boolean | 是否启用 |

### ExternalResourceRegistration (`t_external_resource_registration`)

外部资源注册表，存储从平台发现或手动注册的资源。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| kind | String(32) | 资源类型：external_agent / mcp_server / skill / workflow |
| name | String(255) | 展示名称 |
| description | Text | 资源描述 |
| config | JSONB | 协议相关配置 |
| credentials_ref | String(255) | 凭证引用 |
| is_active | Boolean | 是否启用 |
| platform_registration_id | Integer | 来源平台注册ID（手动注册时为 NULL） |
| external_id | String(255) | 平台侧原生 ID |

### WorkspaceResourceMount (`t_workspace_resource_mount`)

工作空间资源挂载表，将资源关联到具体工作空间。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| workspace_id | Integer | 工作空间 ID |
| resource_type | String(50) | 资源类型：knowledge_base / database / agent_app 等 |
| resource_id | String(255) | 资源 ID |
| mount_config | JSON | 挂载配置 |
| associated_resource_id | Integer | 关联资源 ID（双向关联） |

## 平台适配器

### 架构

采用策略模式 + 工厂模式，通过 `PlatformService.ADAPTER_MAP` 根据平台类型分发到对应适配器。[^src-2026-05-08-外部资源管理功能补全开发记录]

```
PlatformService.discover_platform_resources()
  ↓ 查询 PlatformRegistration
  ↓ 根据 platform_type 查找适配器类
  ↓ 创建适配器实例
PlatformAdapter.discover_resources(kind)
  ↓ 调用外部平台 API
  ↓ 解析返回数据为统一格式
DiscoverResult (added / updated / skipped / errors)
  ↓ 对比已有 ExternalResourceRegistration
  ↓ 新增/更新/跳过
数据库提交
```

### 适配器接口

`core/helpers/platforms/base_adapter.py`:

```python
class PlatformAdapter(ABC):
    async def discover_resources(self, kind: str | None = None) -> DiscoverResult: ...
    def supported_resource_kinds(self) -> list[str]: ...
```

### 已实现的适配器

| 适配器 | 支持资源类型 | API 端点 |
|--------|------------|---------|
| `DifyPlatformAdapter` | external_agent, workflow | `/console/api/apps` |
| `DscPlatformAdapter` | workflow | `/api/workflows` |
| `SkillHubPlatformAdapter` | skill | `/api/skills` |

Dify 适配器通过 app.mode 字段推断资源类型：agent/chat 模式映射为 `external_agent`，其余为 `workflow`。[^src-2026-05-08-外部资源管理功能补全开发记录]

## API 路由

### 平台管理 (`/wp/platforms`)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/wp/platforms?platform_type=` | 平台列表 |
| POST | `/wp/platforms` | 注册平台 |
| PATCH | `/wp/platforms/{id}` | 更新平台 |
| DELETE | `/wp/platforms/{id}` | 停用平台（软删除） |
| POST | `/wp/platforms/{id}/discover` | 触发资源发现 |

### 外部资源管理 (`/wp/external-resources`)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/wp/external-resources?kind=` | 资源列表 |
| POST | `/wp/external-resources` | 创建资源 |
| PATCH | `/wp/external-resources/{id}` | 更新资源 |

### 资源挂载 (`/wp/workspaces`)

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/wp/workspaces/mount-resource` | 挂载资源 |
| POST | `/wp/workspaces/batch-mount-resources` | 批量挂载 |
| POST | `/wp/workspaces/unmount-resource` | 卸载资源 |
| GET | `/wp/workspaces/{id}/mounted-resources` | 获取挂载资源 |
| POST | `/wp/workspaces/associate-resources` | 关联资源 |

## 资源发现去重逻辑

1. 查询已有资源行，按 `external_id` 建立索引
2. 遍历发现结果：
   - external_id 已存在 → 比较名称/描述/配置，有变化则更新，无变化则跳过
   - external_id 不存在 → 新建 ExternalResourceRegistration 行
3. 统一提交事务，确保原子性[^src-2026-05-08-外部资源管理功能补全开发记录]

## 前端架构

```
PlatformConfigPage (入口)
  ├── LLM 配置
  ├── Claw Code
  ├── 领域配置
  ├── 平台注册 → PlatformManagementPage
  └── 平台设置（预留）

ExternalResourcesPage
  ├── 资源列表（按类型筛选）
  ├── 新建注册对话框
  └── 平台管理 → 跳转至 PlatformManagementPage
```

## 安全设计

- 所有凭证不存储明文，仅存储密钥管理系统的引用
- API 调用使用 HTTPS 加密传输
- 平台访问权限集成到 Keycloak 体系
- 所有资源操作记录审计日志
