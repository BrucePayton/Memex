# 外部资源管理模块架构

## 概述

外部资源管理模块是 AskData 平台集成外部服务（Dify / DSC / Skill Hub / MCP 等）的核心组件，分为**外部连接管理**、**资源注册管理**、**资源挂载管理**三个层级。

> **2026-05-09 架构变更**：`PlatformRegistration` 标记为 deprecated，引入 `ExternalConnection` 替代独立"平台注册"概念。平台连接能力直接嵌入外部资源注册流程。[^src-2026-05-09-外部资源管理重构-领域模板CRUD-开发记录]

## 三层架构

```
┌─────────────────────────────────────────────────────┐
│               外部连接层 (External Connection)         │
│  t_external_connection                              │
│  Dify/Coze/n8n/ClawHub 等平台连接与认证信息            │
│  ↓ 通过 connection_id 关联                             │
├─────────────────────────────────────────────────────┤
│               资源注册层 (External Resource)          │
│  t_external_resource_registration                   │
│  external_agent / mcp_server / skill / workflow     │
│  ↓ 通过挂载引用                                       │
├─────────────────────────────────────────────────────┤
│               资源挂载层 (Workspace Mount)            │
│  t_workspace_resource_mount                         │
│  将资源挂载到具体工作空间                             │
│  resource_type: knowledge_base / database /          │
│    agent_app / mcp_server / skill / workflow         │
└─────────────────────────────────────────────────────┘
```

## 数据模型

### ExternalConnection（`t_external_connection`）— 新增 [^src-2026-05-09-外部资源管理重构-领域模板CRUD-开发记录]

外部连接表，替代原 `t_platform_registration`。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| connection_type | String(64) | 连接类型：`dify` / `coze` / `n8n` / `clawhub` |
| name | String(255) | 展示名称 |
| base_url | String(512) | 平台 API 基地址 |
| auth_config | JSONB | 认证配置（加密存储，含用户名/密码/API Key 等） |
| config | JSONB | 扩展配置 |
| is_active | Boolean | 是否启用 |

### ExternalResourceRegistration（`t_external_resource_registration`）— 变更

`platform_registration_id` 列保留为 NULL，不再使用。新增通过 `connection_id` 关联到 ExternalConnection。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| kind | String(32) | 资源类型：`external_agent` / `mcp_server` / `skill` / `workflow` |
| name | String(255) | 展示名称 |
| description | Text | 资源描述 |
| config | JSONB | 协议相关配置 |
| credentials_ref | String(255) | 凭证引用 |
| is_active | Boolean | 是否启用 |

### WorkspaceResourceMount（`t_workspace_resource_mount`）— 扩展

resource_type 新增 `mcp_server` / `skill` / `workflow` 支持。[^src-2026-05-09-外部资源管理重构-领域模板CRUD-开发记录]

### PlatformRegistration — 已废弃

保留表结构和代码（标记 deprecated），不再被前端调用。非破坏性数据迁移。

## 三类资源分类

资源管理页面采用卡片式分类，不再使用扁平的 `kind` tabs：[^src-2026-05-09-外部资源管理重构-领域模板CRUD-开发记录]

| 分类 | 包含 kind | 管理视图 |
|------|----------|----------|
| **Agent / 工作流** | `external_agent` + `workflow` | 连接管理 → 发现应用 → 批量导入 |
| **Skill** | `skill` | Skill Hub 连接 → 发现技能 → 导入 |
| **MCP** | `mcp_server` | 直接配置 MCP Server（SSE/Stdio/Streamable HTTP） |

## 平台适配器

### 架构

采用策略模式 + 工厂模式，通过 `CONNECTION_ADAPTER_MAP` 根据连接类型分发到对应适配器。[^src-2026-05-09-外部资源管理重构-领域模板CRUD-开发记录]

```
POST /external-connections/{id}/discover
  ↓ 根据 connection_type 查找适配器
PlatformAdapter.discover_resources()
  ↓ 调用外部平台 API
  ↓ 解析返回数据为统一格式
DiscoverResult
  ↓ 对比已有 ExternalResourceRegistration
  ↓ 新增/更新/跳过
```

### 已实现的适配器

| 适配器 | 连接类型 | 支持资源类型 | 认证模式 |
|--------|----------|------------|---------|
| `DifyPlatformAdapter` | dify | external_agent, workflow | Console API (用户名/密码) + Bearer Token |
| `SkillHubPlatformAdapter` | clawhub | skill | API Key |
| MCP 配置模型 | — | mcp_server | 无需发现，直接配置 |

**Dify Console API 认证流程**：[^src-2026-05-09-外部资源管理重构-领域模板CRUD-开发记录]
1. POST `/console/api/login` 获取 session cookie
2. 带 cookie 调用 `/console/api/apps` 获取应用列表
3. 按 app.mode 推断资源类型：`agent`/`agent-chat` → `external_agent`，`workflow` → `workflow`

## API 路由

### 外部连接管理（`/wp/external-connections`）— 新增 [^src-2026-05-09-外部资源管理重构-领域模板CRUD-开发记录]

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/wp/external-connections` | 连接列表 |
| POST | `/wp/external-connections` | 创建连接 |
| PATCH | `/wp/external-connections/{id}` | 更新连接 |
| DELETE | `/wp/external-connections/{id}` | 删除连接 |
| POST | `/wp/external-connections/{id}/discover` | 从连接发现资源 |

### 外部资源管理（`/wp/external-resources`）— 扩展

| 方法 | 路径 | 说明 | 变更 |
|------|------|------|------|
| GET | `/wp/external-resources?category=` | 资源列表 | 新增 category 过滤 |
| POST | `/wp/external-resources` | 创建资源 | 新增 connection_id 字段 |
| PUT | `/wp/external-resources/{id}` | 更新资源 | 新增 |
| DELETE | `/wp/external-resources/{id}` | 软删除 | 新增 |
| POST | `/wp/external-resources/batch-import` | 批量导入 | 新增 |

### 平台管理（`/wp/platforms`）— 已废弃

API 端点保留但前端不再调用。

### 资源挂载（`/wp/workspaces`）— 扩展

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/workspaces/{id}/available-external-resources?category=` | 获取可挂载的外部资源 |
| POST | `/wp/workspaces/mount-resource` | 挂载资源（含 mcp_server/skill/workflow） |

## 前端架构

```
ExternalResourcesPage（分类卡片入口）
  ├── Agent / 工作流 → AgentWorkflowView
  │   ├── 连接 CRUD（DifyConnectionDialog）
  │   ├── 应用发现（DiscoverAppsDialog）
  │   └── 已注册列表
  ├── Skill → SkillView
  │   ├── Skill Hub 连接
  │   ├── 技能发现/导入
  │   └── 命令行安装引导
  └── MCP → MCPView
      ├── 模式选择（SSE/Stdio/Streamable HTTP）
      └── 已注册 MCP Server 列表

已移除：PlatformManagementPage + /platforms 路由
```

## 安全设计

- 所有凭证不存储明文，auth_config 通过 `ExternalConnection` 集中加密管理
- API 调用使用 HTTPS 加密传输
- 平台访问权限集成到 Keycloak 体系
- 所有资源操作记录审计日志
