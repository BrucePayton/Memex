## 问题概述

2026-05-08 平台配置模块存在三个优化问题：

1. **入口位置错误**：平台配置通过首页卡片进入，而非侧边栏系统管理菜单
2. **首页冗余入口**：首页核心功能区有 PlatformConfig 卡片入口需移除
3. **权限无法访问**：platform admin 用户在 Keycloak 配置了 `PlatformConfig` 子组，但仍无法进入平台配置模块

---

## Root Cause 分析

### 根因 1：侧边栏隐藏逻辑

**文件**: `web/src/app/routes/routeAccess.ts`

`isSidebarHiddenPageModule` 函数将 `platform-config`、`llm-config`、`claw` 三个 page_id 标记为侧边栏隐藏，导致这些页面不会出现在 MainSide 菜单中。

```typescript
// 原代码：
const PLATFORM_CONFIG_HIDDEN_PAGE_IDS = new Set(['platform-config', 'llm-config', 'claw']);
```

### 根因 2：page_group_defaults.py 缺少平台配置页面组定义

**文件**: `core/modules/user/page_group_defaults.py`

`ASKDATA_PAGE_SUBGROUP_DEFAULTS` 字典只定义 7 个页面组，**完全没有** `PlatformConfig`、`LlmConfig`、`Claw`、`DomainConfig` 的条目。

当 Keycloak 子组 `PlatformConfig` 找不到匹配的默认配置时，`page_permission_service.py` 的 `_merge_subgroup_page_fields` 方法会回退到组名推导：

| 字段 | 推导逻辑 | 结果 |
|------|---------|------|
| `page_route` | `f"/{gname}"` | `/PlatformConfig`（期望 `/platform-config`） |
| `parent_menu` | 默认 `"core_features"` | 显示在核心功能而非系统管理 |
| `page_id` | 从 Keycloak attributes 取 | 可能缺失或不匹配 |

这意味着：
- 前端 ACL 检查 `acl.includes('/platform-config')` 永远为 false，因为后端返回的是 `/PlatformConfig`
- 页面显示在"核心功能"而非"系统管理"

### 根因 3：前端路由权限体系不完整

`PROTECTED_APP_PATHS` 缺少 `LLM_CONFIG`、`DOMAIN_CONFIG`、`CLAW` 等路由；
`isPathAllowedByAcl` 和 `userHasPlatformConfigAccess` 均未纳入 `domain-config`；
`appRouteMaps.ts` 缺少 `domain-config` 的 page_id → route 映射。

## 修复方案

### 修复 1：后端 page_group_defaults.py 新增 4 个页面组

**文件**: `core/modules/user/page_group_defaults.py`

新增条目：

| 页面组 (Keycloak 子组名) | page_id | page_route | parent_menu | sort_order |
|--------------------------|---------|------------|-------------|-----------|
| `PlatformConfig` | `platform-config` | `/platform-config` | `system_management` | 60 |
| `LlmConfig` | `llm-config` | `/llm-config` | `system_management` | 61 |
| `Claw` | `claw` | `/claw` | `system_management` | 62 |
| `DomainConfig` | `domain-config` | `/domain-config` | `system_management` | 63 |

**关键原则**：`parent_menu: 'system_management'` 确保页面显示在侧边栏的"系统管理"分组下，与用户管理（UserMgt）并列。

### 修复 2：前端移除侧边栏隐藏

**文件**: `web/src/app/routes/routeAccess.ts`

- 清空 `PLATFORM_CONFIG_HIDDEN_PAGE_IDS` —  平台配置页面可在侧边栏显示
- 删除 `isSidebarHiddenPageModule` 中对平台配置路由的隐藏判定（`/platform-config`、`/llm-config`、`/claw`）

### 修复 3：前端补充路由权限完整性

| 文件 | 修改内容 |
|------|---------|
| `web/src/app/routes/routeAccess.ts` | `PROTECTED_APP_PATHS` 加入 `LLM_CONFIG`、`DOMAIN_CONFIG`、`CLAW`；`isPathAllowedByAcl` 和 `userHasPlatformConfigAccess` 纳入 `domain-config` |
| `web/src/app/routes/appRouteMaps.ts` | `AppActivePage` 类型加入 `domain-config`；`pageToRouteMap` 和 `routeToPageEntries` 补充映射 |
| `web/src/components/MainSidebar.tsx` | `pageIdToDescription` 补充 `platform-config`、`llm-config`、`domain-config`、`claw` 描述 |

### 修复 4：首页移除平台配置卡片

**文件**: `web/src/features/home/pages/HomePage.tsx`

- 移除 `userHasPlatformConfigAccess` 导入
- 移除 `canAccessPlatformConfig` 变量
- 删除整个平台配置 Card 组件（约 60 行）

### 修复 5：优化平台配置页面返回按钮

**文件**: `web/src/features/platform-config/pages/PlatformConfigPage.tsx`

返回按钮改为 `window.history.length > 2 ? navigate(-1) : navigate(ROUTES.HOME)`，支持从侧边栏进入后返回上一页。

## 数据流

修复后的数据流如下：

```
Keycloak PlatformConfig 子组
  └─ 拥有 admission role 与用户角色交集
      └─ page_permission_service.get_user_accessible_pages()
           ├─ 匹配 page_group_defaults.PlatformConfig 默认配置
           ├─ _merge_subgroup_page_fields() 合并 Keycloak attributes
           └─ 返回 { page_id: "platform-config", page_route: "/platform-config", parent_menu: "system_management", ... }
               └─ 存入 Redis 会话 → 登录响应 accessiblePages[]
                   └─ 前端 AuthContext
                        ├─ MainSidebar.getUserAccessibleMenus()
                        │    ├─ isSidebarHiddenPageModule → false (不再隐藏)
                        │    ├─ parent_menu === "system_management" → 系统管理分组
                        │    └─ 显示菜单项
                        └─ ProtectedRoute.isRouteAllowedForUser()
                             ├─ acl.includes("/platform-config") → true
                             └─ 允许渲染 PlatformConfigPage
```

## 经验总结

1. **前后端页面权限数据流一致是关键**：后端 `page_group_defaults.py` 的 `page_route` 必须与前端 `ROUTES` 常量精确匹配（大小写敏感），否则 ACL 检查失败
2. **`parent_menu` 决定侧边栏分组位置**：需要显示在"系统管理"下必须设为 `system_management`，默认值是 `core_features`
3. **Keycloak attributes 可覆盖默认值但建议设默认**：`page_group_defaults.py` 作为兜底配置可以减少 Keycloak 端的手动配置工作
4. **`PROTECTED_APP_PATHS` 需要与路由配置同步维护**：新增受保护路由必须同步加入该列表，否则路由守卫无法识别
5. **所有入口要统一**：同一功能模块应该只有一个入口方式（卡片或侧边栏），避免多渠道入口导致用户困惑
