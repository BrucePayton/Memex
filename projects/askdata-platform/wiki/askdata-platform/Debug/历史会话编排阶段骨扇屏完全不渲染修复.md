# 修复：历史会话编排阶段（骨扇屏）完全不渲染

## 概述

2026-05-14 修复的 bug：打开历史会话时，API 成功返回了包含 `orchestrator_trace` 的真实历史消息数据，但编排阶段（orchestrator-stage）的内容完全不显示，仅留空白气泡。

## 根因

**文件**：`web/src/features/query-space/hooks/useStageTreeMemo.ts`（第 32-38 行）

`useStageTreeMemo` 在构建 `childStageIds` 集合时，**无论父节点是否存在**都将子阶段加入 `childStageIds`。后端 `combo_executor.py` 发送的阶段事件大量使用 `parent_stage_id="supper_short"`，但父阶段 `stage_id="supper_short"` 本身从未被发送或持久化。渲染逻辑（`QueryAnalysisChat.tsx` 第 6217-6224 行）通过 `childStageIds.has(...)` 判断子阶段，所有子阶段被标记为"子节点"后 `return null` 全部跳过。

## 修复方案

在 `useStageTreeMemo.ts` 中添加父阶段存在性检查：

```diff
- if (pid) {
+ if (pid && stageMessagesById.has(`${tk}::${pid}`)) {
```

这与同文件中 `buildStageForest` 函数（第 666 行）已有的正确模式保持一致：父节点不存在时将子节点提升为根节点。

## 涉及文件

| 文件 | 作用 |
|------|------|
| `web/src/features/query-space/hooks/useStageTreeMemo.ts` | 阶段树构建 — 修复位置 |
| `web/src/features/query-space/pages/QueryAnalysisChat.tsx` | 渲染逻辑 — 受影响位置 |
| `core/executor/combo_executor.py` | 后端阶段事件发送 — 根因侧 |

## 教训

1. 父子关系映射必须验证父节点是否存在
2. `buildStageForest` 已有正确实现模式，`useStageTreeMemo` 应保持一致
3. 流式响应和历史加载使用不同的阶段构建逻辑，需要统一验证
