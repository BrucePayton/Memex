# 问数空间资源管理页面初始化请求风暴修复

## 问题背景

在 AskData Platform 中，创建或选择问数空间后点击"资源管理"，页面瞬间发起大量 API 请求，形成"请求风暴"。

- **现象**：点击"资源管理"按钮后，浏览器 Network 面板显示 7+ 个请求同时发出
- **影响**：服务器瞬时压力增大，页面加载响应变慢，部分请求可能被取消或超时
- **触发条件**：任何进入 `QuerySpaceResourceManagement` 组件的场景（新建空间、切换空间）

## 问题分析

### 技术根因

1. **初始化 useEffect 过度加载**（`QuerySpaceResourceManagement.tsx` init useEffect）

   当 `selectedQuerySpace?.id` 变化时，一次性触发所有数据加载：
   - `refreshMountedResources()` — 已挂载资源（1 req）
   - `loadKnowledgeBases()` — 知识库（1 req）
   - `loadDatabases()` — 数据库（1 req）
   - `getAnalysisReadAppsCatalog()` — 分析目录（1 req）
   - `loadExternalResources()` — 外部资源 × 4（4 req，含重复）

   共 **7+ 请求同时发出**，而用户默认看到的是"已挂载资源"tab，大多数数据根本不需要。

2. **重复的 agent_workflow 请求**（copy-paste bug）

   ```typescript
   // 旧代码 — 连续两行相同的请求
   getExternalResources({ category: 'agent_workflow' }),
   getExternalResources({ category: 'agent_workflow' }),  // 重复！
   getExternalResources({ category: 'mcp' }),
   getExternalResources({ category: 'skill' }),
   ```

3. **标签切换 useEffect 与初始化联动**

   标签切换 useEffect 的依赖为 `[currentTab, selectedQuerySpace?.id, activeResourceTab]`，
   当 `selectedQuerySpace?.id` 变化时，**两个 useEffect 同时触发**，导致额外重复加载。

### 请求时序

```
选中间 → init useEffect 触发
  ├─ GET /wp/workspaces/{id}/mounted-resources   (1)
  ├─ GET /wp/workspaces/{id}/knowledgebases       (1)
  ├─ GET /wp/workspaces/{id}/datasources          (1)
  ├─ GET /wp/analysis-read/apps/catalog           (1)
  ├─ GET /wp/external-resources?category=agent_workflow  (2, 重复!)
  ├─ GET /wp/external-resources?category=mcp      (1)
  └─ GET /wp/external-resources?category=skill    (1)
```

## 修复方案

### 文件

`web/src/components/QuerySpaceResourceManagement.tsx`

### 1. 缩减 init useEffect 范围

初始化只加载默认 tab 需要的数据（已挂载资源），其余全部按需懒加载：

```typescript
// 仅刷新已挂载资源
refreshMountedResources();
// 重置外部资源加载标记，切换空间后需要重新加载
loadedExternalResourcesRef.current = false;
```

知识库、数据库、分析目录、外部资源均移至标签切换 useEffect 中按需加载。

### 2. 修复重复请求

将 4 个请求精简为 3 个，从 `agent_workflow` 的结果中按 `kind` 拆分：

```typescript
const [agentWfRes, mcpRes, skillRes] = await Promise.all([
  getExternalResources({ category: 'agent_workflow' }),
  getExternalResources({ category: 'mcp' }),
  getExternalResources({ category: 'skill' }),
]);
const allData = (agentWfRes.data || []);
setExternalResources({
  external_agent: allData.filter(r => r.kind === 'external_agent'),
  workflow: allData.filter(r => r.kind === 'workflow'),
  mcp_server: mcpRes.data || [],
  skill: skillRes.data || [],
});
```

### 3. 外部资源懒加载 + 缓存

- 仅在用户切换到 `workflows` / `mcp` / `skills` 子 tab 时加载
- 使用 `loadedExternalResourcesRef` 标记避免重复请求
- 切换工作空间时重置标记，确保获取新空间的数据

### 4. 修复标签切换 useEffect 依赖

移除 `selectedQuerySpace?.id` 依赖，避免与初始化联动：

```typescript
// 修复前
}, [currentTab, selectedQuerySpace?.id, activeResourceTab]);

// 修复后
}, [currentTab, activeResourceTab]);
```

使用 `useRef` 替代 `sessionStorage` 追踪前一标签，避免 I/O 副作用。

## 修复效果

| 场景 | 修复前 | 修复后 |
|------|--------|--------|
| 点击"资源管理" | 7+ 请求 | **1 请求**（仅已挂载资源） |
| 切换到 可用资源 > 知识库 | 0（已加载） | **1 请求**（按需） |
| 切换到 可用资源 > Skill | 0（已加载） | **3 请求**（按需，缓存后不再请求） |

## 最佳实践

### React 组件数据加载原则

1. **按需加载优先** — 只加载用户当前可见 tab 的数据，不要预加载所有 tab
2. **useEffect 职责单一** — 一个 useEffect 只做一件事，避免多 useEffect 因同一依赖联动
3. **去重标记** — 对昂贵的外部 API 调用使用 `useRef` 标记是否已加载
4. **依赖数组精准** — 只包含真正需要触发的变量，避免将 `selectedXxx?.id` 放入不相关的 useEffect

### 重复请求排查方法

- 浏览器 DevTools → Network → 按名称排序，查找重复 URL
- 检查 `Promise.all` 数组是否有 copy-paste 产生的重复调用
- 注意 API 层的 `dedupe` 选项只对**同一时刻**的相同请求有效，连续两次调用仍会发出两次
