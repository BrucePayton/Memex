# 外部资源管理功能补全开发记录 2026-05-08

## 问题背景
外部资源管理功能的平台注册管理部分在设计文档（`外部资源管理功能设计与实现.md`）中有完整的 API 设计，但后端 API 端点和前端页面均未实现。同时 `core/helpers/platforms/` 目录下只有 `.pyc` 编译文件（dify_client、dsc_client、skill_hub_client），`.py` 源文件未提交到 git。

## 发现的问题

### 1. 平台注册 API 未实现
- 设计文档中描述了 `GET/POST/PATCH/DELETE /wp/platforms` 和 `POST /wp/platforms/{id}/discover`
- 但 `core/modules/workspace/views.py` 中无对应路由
- `core/modules/workspace/request_model.py` 中无对应的 Request Model

### 2. 平台适配器源代码缺失
- `core/helpers/platforms/` 下只有 `__pycache__/` 中的编译文件
- `dify_client.py`、`dsc_client.py`、`skill_hub_client.py` 的 `.py` 源文件不存在
- 无 `__init__.py` 和 `base_adapter.py`

### 3. 前端平台管理页面缺失
- wiki 中描述的 `PlatformManagementPage.tsx` 未实现
- 前端 API 封装 `apis/workspace/platforms.api.ts` 不存在
- `PlatformConfigPage.tsx` 中"平台设置"模块为 disabled 状态

### 4. 资源类型定义不一致
- DB 模型支持 4 种 kind（external_agent/mcp_server/skill/workflow）
- 但 API 和前端只定义了 3 种（external_agent/mcp_server/dsc_workflow）
- 缺少 skill 和 workflow 类型的前端支持

### 5. 路由冲突
- 旧的 `/platforms` 路由被重定向到 `/external-resources`（兼容历史书签）
- 新平台管理页面也需要 `/platforms` 路由

## 解决方案

### 后端补全

#### Request Model (`core/modules/workspace/request_model.py`)
- 新增 `ExternalResourceCreateRequest` 的 kind 支持 skill/workflow
- 新增 `PlatformCreateRequest`（platform_type, name, base_url, api_key_ref, config）
- 新增 `PlatformPatchRequest`（所有字段可选）
- 新增 `PlatformDiscoverRequest`（可选 kind 过滤）

#### API Views (`core/modules/workspace/views.py`)
- 导入 `PlatformRegistration` 模型和新的 request model
- 新增 5 个路由：
  - `GET /wp/platforms` — 平台列表，支持 platform_type 过滤
  - `POST /wp/platforms` — 注册平台
  - `PATCH /wp/platforms/{platform_id}` — 更新平台（含软删除）
  - `DELETE /wp/platforms/{platform_id}` — 停用平台
  - `POST /wp/platforms/{platform_id}/discover` — 触发平台资源发现

#### 平台服务 (`core/modules/workspace/platform_service.py`)
- `PlatformService` 类，包含 `ADAPTER_MAP` 映射平台类型到适配器类
- `discover_platform_resources` 方法：
  1. 查询 PlatformRegistration 行
  2. 根据 platform_type 创建对应适配器
  3. 调用适配器的 discover_resources
  4. 对比已有 ExternalResourceRegistration（按 external_id 去重）
  5. 新增/更新/跳过资源
  6. 返回 {added, updated, skipped, errors}

### 平台适配器补全

#### 基类 (`core/helpers/platforms/base_adapter.py`)
- `PlatformAdapter` 抽象基类，定义 `discover_resources` 和 `supported_resource_kinds` 接口
- `DiscoverResult` 数据类，包含 added/updated/skipped/errors/resources 字段
- `_build_headers` 辅助方法构建 Authorization header

#### Dify 适配器 (`core/helpers/platforms/dify_client.py`)
- 调用 Dify Console API `/console/api/apps` 获取 App 列表
- 根据 app.mode 推断 resource_kind（agent/chat → external_agent，其余 → workflow）
- 返回 DiscoverResult 包含资源元数据

#### DSC 适配器 (`core/helpers/platforms/dsc_client.py`)
- 调用 DSC API `/api/workflows` 获取工作流列表
- 仅支持 workflow 资源类型
- 返回列表/字典两种响应格式的兼容处理

#### Skill Hub 适配器 (`core/helpers/platforms/skill_hub_client.py`)
- 调用 Skill Hub API `/api/skills` 获取技能列表
- 仅支持 skill 资源类型
- 支持 slug/id/skill_id 多种 ID 字段兼容

### 前端补全

#### API 封装 (`web/src/apis/workspace/platforms.api.ts`)
