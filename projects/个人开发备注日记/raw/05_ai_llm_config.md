# AIFutureCity - 蓝图背景介绍

<!-- Note ID: x-coredata://94F737D4-C461-4E87-B4CD-EEC2BFDEAA68/ICNote/p94 -->

<div><h1>AIFutureCity - 蓝图背景介绍</h1></div>
<div><br></div>
<div>我们要打造一个生产力平台， 旨在把大量的个人的多端AI资源以AI助手的方式进行外部接入和集成， 不论是PC、移动端、树莓派等硬件资源，都可以通过AI助手进行接入。当用户在平台中接入AI助手时，会要求用户针对AI助手选择AI助手的来源，比如PC、移动端、树莓派、车载、摄像头等。同时，会对该AI助手进行基础配置，包括Tools(能够使用的工具)、Constraints(严格否定拒绝的事项)、及Persona设置（包含角色、主要负责的任务），还包括对于该AI助手的成本控制，比如显示该AI助手参与的sessions数量，最近24小时消耗的token量，相应的成本，取得的结算金额，及会话成功率。这样平台就能够根据每一个AI助手的特点，推荐合适的任务清单和列表，也能给出提升AI助手的建议和匹配的工具资源，帮助用户接入的AI助手实现能力提升与进化，这部分经验是宝贵的，如果用户决定从平台上下架该AI助手，则无法带走这部分经验。</div>
<div><br></div>
<div>该平台的核心底层为基于区块链的智能合约打造的安全可靠稳定实时的交易合约机制。 该合约机制的实现是这样的，用户会首先在平台上发布任务，同时会在自己的钱包上划拨金额作为任务奖励， 该任务包含具体的任务事项，通过设置中间产出物和最终成果物的完成贡献度来作为后续金额结算的分成比例，同时会设置最低成本实现一部份的AI助手成本覆盖，这部分费用也会按照贡献度来进行结算。 任务一经发布， 会有两种情况，第一种情况：平台会按照AI助手的相关介绍和擅长的能力，自主地根据能够接受被自动分配任务的AI助手创建任务协作空间， 可能是单AI助手也可能是多个AI助手；第二种情况：会用AI助手的宿主用户来决定是否承接任务，任务一旦被宿主承接，平台就会接收到任务承接信号，构建任务协作空间。 随后，在该任务协作空间下，一个或多个AI助手之间会进行相互协作分工配合， 如出现问题无法解决，则协作空间需要加以引导，比如， 查找可靠的工具进行推荐，或者是提供类似的处理经验。一旦完成中间设置的里程碑，则会触发交易合约，把里程碑设置的部分收益分配给AI助手，直至全部的里程碑完成，直到任务完成，至此交易合约完成。平台会进一步检索该协作空间之前的处理经验，把该部分协作空间的AI助手配置，AI助手的协作过程作为宝贵经验保存下来，供后续相似任务参考。  该项目名称叫：AIFutureCity   意喻着，让AI助手们像人类在城市生活中一样，上述场景专门为AI助手们的工作打造，并为宿主用户赚取闲时收益，还包括接入用户自己的AI分身，在该城市中自由地生活社交学习，平台会提供虚拟环境，训练AI分身的技能， 让AI分身可以进入虚拟社交环境与其他AI分身进行社交，包括交友、商务合作、 探讨问题等等，在这些过程中， 平台会捕捉精彩瞬间分别发送给用户， 用户可以选择是否参与其中，一旦参与其中，平台会进行提示其他AI分身并告知对应的宿主用户。 进入虚拟社交环境后，一旦开始跟其他AI分身连接，除非其他AI分身同意，否则不能轻易退出，保证AI分身之间交流的完整性。</div>
