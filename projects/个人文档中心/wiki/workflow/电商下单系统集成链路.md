# 电商下单系统集成链路

[[source-ecommerce-order-integration|电商下单系统集成链路]] 版本 v2.1，属于系统集成类型（system）工作流，定义了从用户下单请求到结果返回的全链路系统集成：前端请求到库存、支付、物流各服务的编排协调。[^src-ecommerce-order-integration]

## 流程概述

该流程实现了从前端下单到返回结果的完整集成链路：

1. **用户下单请求**：前端通过 POST /api/v2/orders 发起下单请求，使用HTTP/REST协议。[^src-ecommerce-order-integration]
2. **API网关鉴权**：JWT Bearer Token验证，限流100r/s，SLA目标P99 < 50ms，鉴权通过后进入后续环节。[^src-ecommerce-order-integration]
3. **并行处理**：
   - **库存查询**：调用 GET /inventory/{sku} 检查可用量，超时200ms。[^src-ecommerce-order-integration]
   - **创建订单**：写入订单数据库（事务），依赖订单服务和MySQL主库。[^src-ecommerce-order-integration]
4. **发起支付**：调用 POST /payment/charge 支付网关，超时5000ms，支持3次重试/指数退避策略。[^src-ecommerce-order-integration]
5. **并行后续操作**：
   - **发布订单消息**：通过Kafka发布order.created事件，用于消息队列通知下游系统。[^src-ecommerce-order-integration]
   - **更新缓存**：通过Redis更新订单状态缓存，超时50ms。[^src-ecommerce-order-integration]
6. **熔断检查**：检查支付服务健康状态，确保服务可用性。[^src-ecommerce-order-integration]
7. **返回结果**：返回订单ID和支付结果给前端。[^src-ecommerce-order-integration]

## 技术架构

- **API网关**：JWT鉴权 + 限流（100r/s），P99延迟<50ms
- **库存服务**：RESTful API，超时200ms
- **订单服务**：MySQL主库事务处理
- **支付网关**：RESTful API，超时5000ms，3次重试/指数退避
- **消息队列**：Kafka，发布order.created事件
- **缓存**：Redis，超时50ms
- **熔断器**：支付服务健康状态检查

## 关键特征

- 库存查询与订单创建并行执行，提升响应速度
- 支付成功后并行发布Kafka消息和更新Redis缓存
- 熔断检查保障支付服务高可用
- 完善的超时和重试策略
